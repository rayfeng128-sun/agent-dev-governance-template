# Harness Quickstart

本文件是 agent 和开发者进入当前仓库的默认轻量入口。除非任务触发完整闭环升级条件，优先阅读本文件即可开始判断范围。

## 当前阶段

- 项目阶段：`prototype-to-code`。
- 交付重点：把<项目名称>原型落成一个可真实使用、可持续扩展的前后端分离系统。
- 默认策略：真实主链优先、正式契约优先、可用闭环优先。

## 主线技术栈

- 前端主线：`frontend/`，<前端技术栈>。
- 后端主线：`backend/`，<后端技术栈 API>，开发环境可使用 demo profile 与 seed/fixture 数据承载测试数据，但实现目标仍是正式系统能力。
- 回退路径：`app/` 原生前端与 `server/` Node 服务仅作为 legacy fallback。

## 数据驱动阶段说明

- `Mock 数据`：
  只用于未完成接口、复杂外部依赖或高风险能力的短期过渡，不得长期作为业务真相来源。
- `demo seed / fixture`：
  用于本地开发、测试和冒烟环境承载可重复的数据样本；它是测试数据来源，不是“只做演示”的理由。
- 实务上可以并存：
  但任何已纳入当前交付范围的核心主链，都应优先落到真实前后端交互、真实状态流转和可验证结果上。

## 需求定义默认姿势

- 默认按用户故事切分需求，而不是按单个原型页面切分。
- 推荐先写清：谁、在什么场景下、想完成什么、获得什么价值。
- 原型页用于说明界面、交互和信息结构，不作为默认需求切分单位。
- 一个需求应尽量形成最小可用闭环，例如“创建考试计划并在列表中看到草稿且刷新后仍存在”，而不是“完成创建页 UI”。

## 流程判定

### 默认使用轻量流程

以下任务默认只需要 `change.md`：

- 页面表现、布局、样式、文案调整。
- 前端壳子优化、非业务性静态信息调整。
- 小型共享组件抽取，不改变业务状态流转。
- 纯文档、模板或治理入口优化。

轻量需求也应使用用户故事表达目标与验收，原型来源仅作为辅助说明。

`change.md` 适合记录以下边界内的需求：

- 可以描述成一个低风险、可独立验证的小故事或故事切片。
- 不需要冻结正式 API 契约、数据库结构或复杂权限规则。
- 即使使用测试数据承载，也不改变正式后端契约。
- 目标更偏“小范围安全变更”，而不是“只把一个故事演起来”。

轻量记录路径：

```text
references/Harness/changes/<version>/<change-id>/change.md
```

### 升级完整闭环流程

命中以下任一条件时，必须阅读完整 `AGENTS.md` 与相关 `rules/`，并补齐对应产物：

- 新增或修改后端 API：补 `api-contract.md`。
- 新增或修改数据库或持久化结构：补 `db-migrations.sql` 与 `rollback.sql`。
- 修改多页面流程、权限、角色、登录态、关键操作准入或核心状态流转：补 `flow.md` 与 `impact.md`。
- 发布级交付或架构调整：补完整十文件闭环。

完整闭环需求默认要求写明用户故事、端到端主路径、故事边界和可独立验收标准。

不要只用 `change.md` 的情况：

- 新增或修改正式后端 API 契约。
- 新增或修改数据库或持久化结构。
- 修改权限、角色、登录态或关键操作准入。
- 修改多页面主流程或核心状态流转。
- 需求无法说成一个最小可用故事闭环，只能说成“完成某页”或“补一个局部界面”。

## 常用验证命令

按变更范围选择最小有效验证：

```bash
npm test
npm --prefix frontend run build
mvn -f backend/pom.xml -Dmaven.repo.local=.m2 test
```

治理规则、模板或 Node 测试变更优先运行 `npm test`。前端 Vue 变更至少运行 `npm --prefix frontend run build`。后端 Spring 变更至少运行 Maven 测试。

## 交付答复要求

最终答复必须包含：

- 本次采用“轻量流程”还是“完整闭环流程”。
- 修改范围摘要。
- 实际运行的验证命令和结果。
- 若未运行某项验证，说明原因。

## 计划维护要求

- `PLANS.md` 是当前阶段的执行看板，不只是导航页。
- 当任务完成导致状态、阶段顺序、完成度或下一步发生变化时，必须同步更新 `PLANS.md`。
- 当新需求已经写好，且用户要求“更新计划”时，必须同步更新 `PLANS.md`，把新需求纳入阶段计划、优先级或后置判断。
